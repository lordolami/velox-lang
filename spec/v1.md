# Velox v1 Language Specification

Version: 1.0.0-draft
Status: Draft (implementation-aligned)
Last Updated: 2026-04-12

## 1. Scope

This spec defines the v1 surface for `.vx` source files compiled by Velox to:
- JavaScript modules (required)
- Embedded WebAssembly for `@fast` functions (required for supported signatures)

This document is implementation-aligned with the current compiler/runtime in `src/`.

## 2. Lexical Grammar

### 2.1 Token Types

- Keywords: `component`, `render`, `fast`
- Structural: `(` `)` `{` `}` `:` `,` `=` `->` `~` `@`
- Literals: `number`, `string`
- Names: `identifier`
- Operators in `symbol` channel: `+ - * / % < > <= >= == != && || !`
- End marker: `eof`

### 2.2 Comments and Whitespace

- Line comments use `//` and run to newline.
- Spaces, tabs, carriage returns, and newlines are ignored except for token positions.

## 3. Top-Level Grammar

Informal grammar:

```text
program        -> declaration* EOF
declaration    -> importDecl | componentDecl | fastDecl
importDecl     -> "import" (string | importClause "from" string) ";"?
importClause   -> defaultImport
               | namespaceImport
               | namedImports
               | defaultImport "," namespaceImport
               | defaultImport "," namedImports
componentDecl  -> "component" IDENT paramList? "{" componentBody* "}"
componentBody  -> stateDecl | renderDecl
stateDecl      -> "~" IDENT "=" expression
renderDecl     -> "render" rawBlock
fastDecl       -> "@" "fast" IDENT typedParamList "->" IDENT rawBlock
```

## 4. Imports

Supported forms:

```vx
import "./side-effect.vx"
import { A, B as C } from "./mod.vx"
import X from "./mod.vx"            // parsed, rejected in project checks for .vx modules
import * as NS from "./mod.vx"
import { Chart } from "chart.js"    // package import allowed
```

Import resolution rules:
- Relative `.vx` imports are resolved and rewritten to `.js` in output.
- Relative extensionless imports resolve as `.vx` at compile time and emit `.js`.
- Relative side-effect CSS imports are supported (`import "./styles.css"`) and copied to output.
- Relative `.js` import in `.vx` source is invalid (`VX3021`).
- Unsupported relative extensions are invalid (`VX3024`).
- Relative import escaping project root is invalid (`VX3023`).
- Package imports are allowed for non-relative specifiers; symbol validation is skipped.

## 5. Components

Example:

```vx
component Counter(label, onInc) {
  ~count = 0
  render {
    <button on:click={onInc}>{label}: {count}</button>
  }
}
```

Rules:
- Exactly one `render { ... }` block is required per component.
- Zero or more `~state` declarations are allowed.
- Duplicate render block: `VX4001`
- Missing render block: `VX4002`

## 6. Expressions (State Initializers)

Supported expression forms:
- Number/string/boolean literals
- Array literals (for example `[1, 2, total]`)
- Object literals (for example `{ name: "Ada", active: true }`)
- Identifiers
- Member/index access (for example `profile.name`, `items[0]`)
- Unary: `-`, `!`
- Binary: `+ - * / % == != < > <= >= && ||`
- Call expressions: `fn(a, b)`
- Parenthesized expressions

Validation includes:
- Unknown identifiers/callables (`VX3004`)
- Operand type mismatch (`VX3005`)
- Self-reference in initializer (`VX3006`)
- Forward state reference (`VX3007`)
- Imported component call in state expression (`VX3003`)
- Imported `@fast` call arity/type checks (`VX3001`, `VX3002`)

## 7. Template Syntax (`render`)

Supported template features:
- Text segments
- Bindings: `{name}`
- Conditionals: `{#if cond}...{:else}...{/if}`
- Loops: `{#each items as item}` and `{#each items as item, i}`
- Event directives: `on:event={handler}`

Template diagnostics:
- `VX4101`-`VX4112` for malformed directives/bindings.

## 8. `@fast` Functions and Wasm

Example:

```vx
@fast add(a: i32, b: i32) -> i32 {
  let c = a + b
  return c
}
```

Current v1 constraints:
- Params: `i32` only
- Return type: `i32` only
- Supports arithmetic/comparison/logical expressions and `let`, local assignment, `if/else`, `while`, `return`

Compilation contract:
- Compiler emits embedded Wasm bytes and JS wrapper.
- Wrapper exports same function name and forwards arguments.
- Unsupported constructs fail at compile time with explicit errors.

## 9. Runtime Contract

### 9.1 Component Factory Return Shape

Each compiled component returns:
- `state`
- `render(): string`
- `events`
- `mount(target): () => void`
- `subscribe(listener): () => void`
- `set(patch): void`

### 9.2 Reactive State

Reactive runtime uses:
- `Proxy`-based state mutation tracking
- Microtask-batched subscriber flush (`queueMicrotask`)
- Patch updates through `set({...})`

### 9.3 Event Binding

Runtime binds template event directives to DOM nodes using generated `data-vx-on-*` attributes and invokes handlers with `(event, state)`.

## 10. Router/Framework Conventions

File-based routes are inferred from `pages/`:
- `pages/index.vx` -> `/`
- `pages/blog/[id].vx` -> `/blog/:id`
- `pages/docs/[...rest].vx` -> `/docs/*rest`
- Route groups `(group)` do not appear in URL path

Special files:
- `_layout.vx` for root/segment layout composition
- `404.vx` for not found
- `*.loading.vx` loading boundary for sibling page
- `*.error.vx` error boundary for sibling page
- `*.data.js` route data module for sibling page

Orphan route support files error:
- `VX3026` loading without page
- `VX3027` error without page
- `VX3028` data without page

## 11. Build Artifacts

Project build can emit:
- Compiled module graph (`.js`)
- Route runtime (`__velox_router.js`) and `index.html` when routing is enabled
- Copied `public/` assets
- Copied route data modules (`*.data.js`)
- `velox-manifest.json` with modules, route metadata, and copied files

## 12. Error Code Domains

- `VX200x`: module/import graph and project-level constraints
- `VX300x`: semantic checks and expression/type constraints
- `VX400x`: top-level and component structural parse errors
- `VX410x`-`VX411x`: template directive/binding parse errors
- `VX420x`: import syntax parse errors
- `VX430x`-`VX433x`: declaration/params/expression parse errors

Detailed codes and fixes are in `spec/diagnostics.md`.
